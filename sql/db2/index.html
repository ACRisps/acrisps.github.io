概念模型

  • 码（key）：唯一标识实体的属性集

  • 联系：一对一、多对多、一对多
  • E-R图

关系的完整性

  • 实体完整性

    • 某一属性组的值能唯一标识一个元组，而其子集不能——候选码，选定一个作为主码。候选码里的属性为主属性。

    • 主属性不能取空（NULL）

  • 参照完整性

    • 外码
    • 属性（组）F是外码，与关系S的主码对应，则F上：
      • 或者为空（NULL）
      • 或者为S中的主码值

  • 用户定义的完整性

    • 例如某个属性要取唯一值等

    • 由用户设置

    • not null、unique

    • CHECK

    • 完整性约束命名子句

范式

  • 多种数据依赖

    • 函数依赖
      • 由学号确定姓名，y = f(x) 的关系；x一样，y一定一样：x -> y；x称为决定因素
      • x -> y && x 的任意真子集 -\-> y，那么y完全函数依赖于x；与之相对是部分函数依赖
    • 多值依赖
      • 有这样一个关系 <仓库管理员，仓库号，库存产品号> ，假设一个产品只能放到一个仓库中，但是一个仓库可以有若干管理员，那么对应于一个 <仓库管理员，库存产品号>有一个仓库号，而实际上，这个仓库号只与库存产品号有关，与管理员无关，就说这是多值依赖。

  • 码

    • 见“关系的完整性”

  • 范式

    • 低级范式模式分解可以得到高级范式 ，这个过程叫规范化

  • 1NF

    • 满足每个分量不可分即可

  • 2NF

    • 在1NF的基础上：非主属性完全依赖于任何一个候选码
    • 如（学号，专业，课程号，分数）中，码为（学号，课程号），专业并不依赖于课程号，不符合2NF。
    • 若不符合2NF
      • 插入异常：没选课没法插入学生信息
      • 删除异常：退选课把自己也删了
      • 修改复杂：若多选课，修改专业时需要修改多行
    • 如何修改：投影分解，将不是完全函数依赖的分离开

  • 3NF

    • 非主属性既不部分依赖，也不传递依赖于码
    • （学号，专业，院楼）中，专业依赖于学号，院楼依赖于专业，间接依赖于学号，不符合3NF。
    • 不符合3NF会有类似2NF的问题，如改专业了，同时修改（专业，院楼）

  • BCNF

    • 每一个决定因素都包含码
    • 即，依赖于码的同时，不能依赖于非码（不包含码的）

  • 4NF

    • 不允许非平凡非函数依赖的多值依赖

事务 检查点恢复

  • 要么不做，要么全做

  • ACID

    • 原子性：不做、全做
    • 一致性：没有做到不做或全做会导致数据库失去一致性
    • 隔离性：事务间不能相互干扰
    • 持续性：永久性，一旦提交，永久更改

  • 故障

    • 事物内部故障
      • 恢复：逆序看log，反向操作该事务所有操作
    • 系统故障
      • 恢复：正向看log，已提交的重做（重复执行log），未完成的撤销
    • 介质故障
      • 恢复：重装-重做
    • 计算机病毒

  • 数据转储

    • 装入后备副本，重新运行运行转储后所有更新事务
    • 静态转储：停数据库，转储
    • 动态转储：转储、用户事务并发 + 日志文件

  • 日志文件

    • 记录事务对数据库更新操作的文件
    • 包括：开始BEGIN TRANSACTION标记、COMMIT|ROLLBACK结束标记、更新操作
    • 作用：
      • 事务故障恢复
      • 系统故障恢复
      • 动态转储必需
      • 由转储状态到故障前状态，不必运行已完成的事务程序
    • 严格按时间顺序；先写日志后写数据库

  • 检查点

    • 记录了：检查点建立时所有活动的事务以及它们日志的地址

    • 重新开始文件记录了检查点记录在日志里的地址

并发

  • 封锁
    • 先加锁，再操作：排他锁-写锁、共享锁-读锁
    • IS、IX、SIX（读，写子）
  • 时间戳、乐观锁、多版本并发控制

NoSQL

  • 阻抗失谐：关系模型和内存中的数据结构之间存在差异
  • SQL有强一致性，不便于大规模分布式集群
  • 分布式的非关系型数据库
  • 仲裁

    • 写入仲裁(write quorum)：

      • 如果发生两个相互冲突的写入操作，那么只有其中一个操作能为超过半数的节点所认可，W>N/2 。即，参与写入操作的节点数(W)，必须超过副本节点数(N)的一半。副本个数又称为“复制因子（对等式分布模型）
      • 只需要向主节点中写入数据（主从式分布模型）

    • 读取仲裁

      • 只有当R+W>N时，才能保证读取操作的“强一致性”。其中，执行读取操作时所需联系的节点数(R)，确认写入操作时所需征询的节点数(W)，以及复制因子(N)（对等式分布模型）
      • n只需从主节点中读取数据（主从式分布模型）
